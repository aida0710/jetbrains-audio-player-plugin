# Audio Player プラグイン 使いやすさ向上 設計仕様

作成日: 2026-06-16

## 背景

`Audio Player` は JetBrains IDE 上で音声ファイルを再生する Kotlin 製プラグイン。
現状の主な機能:

- 再生 / 一時停止 / 停止 / シーク / 音量 / ループ、Space キーで再生切替
- ファイル情報テーブル（エンコーディング・フォーマット・チャンネル・サンプルレート・サイズ・再生時間）
- 波形（ボタンで生成）／スペクトラム（開くと自動生成）の画像表示
- ffmpeg/ffprobe の自動検出＋設定画面、未検出時はリンク表示
- タブ切替時の自動一時停止

再生エンジンは `javax.sound.sampled.Clip`（ffmpeg で WAV に変換 → Java で再生）。
シークは `clip.microsecondPosition`、再生位置は `clip.microsecondPosition` で取得できる。

## ゴール

本仕様は「使いやすさ向上」を 1 つのまとまりとして扱い、以下の 5 点を実現する。

- **A. 波形・スペクトラムのインタラクティブ化**
  - 開いた時に波形を自動生成して表示（現在は手動ボタン）
  - 画像上に再生位置の縦ラインを描画（既存の 100ms タイマーで更新）
  - 画像クリックでその位置へシーク（波形・スペクトラム共通。横軸が時間のため両方で機能）
- **B. 再生操作の強化**
  - ←/→ キーで −5 秒/+5 秒シーク、Home キーで先頭へ（Space は既存のまま）
  - 音量・ループ状態をセッションをまたいで記憶し、次に開いた時に復元
- **C. 初回セットアップの分かりやすさ**
  - ffmpeg/ffprobe 未検出のままファイルを開いた時にバルーン通知を表示。「設定を開く」アクション＋ OS 別インストールコマンドの案内。現在の設定リンクは併存
- **D. IDE 分割でも自由にリサイズ可能に**
  - エディタ分割（左にテキスト／右にプレビュー）でも、プラグイン内部の分割でも、最小幅・最小高の「壁」をなくす
- **E. ビジュアライザの表示/非表示トグル**
  - 不要なときは下半分の可視化パネルを丸ごと消し、情報＋コントロールだけのコンパクト表示にできる。状態を記憶

## 非ゴール（今回見送り）

- **再生速度の変更**: 現在の `Clip` 方式では音程を保った速度変更ができず、別の再生経路（ffmpeg `atempo` 再エンコード等）が必要。コストに対し効果が小さいため見送り
- **複数ファイルのナビゲーション**（前後の曲へ移動／連続再生／プレイリスト）: 今回のスコープ外

## リサイズの「壁」の原因と対策（D）

### 原因

現在 `imageLabel`（波形/スペクトラム）は 800×200px の画像をそのまま保持する `JLabel` で、
スクロールも縮小もしない。Swing では画像入り `JLabel` の最小サイズが画像サイズそのものになるため、
パネル全体の最小幅が約 800px に固定される。IDE のエディタ分割で右側を狭めようとすると、
この最小幅が「壁」になる。プラグイン内部の `JSplitPane` も、子要素の最小サイズ合計より小さくできない。

### 対策

- 新設する `TimelineImagePanel` を「利用可能な幅に合わせて画像を拡縮し、最小サイズを小さく申告する」設計にする
- プラグイン内部の各 `JSplitPane` の子要素にも小さい最小サイズを設定し、自由に縮められるようにする

## 採用するアプローチ

画像の上に再生位置ラインを描き、クリックでシークする部分が中心。

- **採用: 専用コンポーネント `TimelineImagePanel` を新設**
  現在の静的な `JLabel`（`imageLabel`）を、`paintComponent` で「画像＋再生位置の縦ライン」を描く独自 `JComponent` に置き換える。クリック位置→時間の変換とシークもこのコンポーネントが担当。波形・スペクトラム両方で再利用できる。Swing の標準的な作法で、リサイズや再描画も素直。最小サイズを小さく申告できるため要件 D も同時に解決する。
- **不採用**: JLabel に重ねる透明グラスペイン方式 / 100ms ごとに画像へ線を焼き込む方式（複雑・無駄が多く利点がない）

## コンポーネント設計

| ファイル | 変更 | 内容 |
|---|---|---|
| `TimelineImagePanel.kt` | 新規 | `BufferedImage?` ＋ 総時間(`durationMicros`) ＋ 現在位置(`positionMicros`) ＋ `onSeek: (Long) -> Unit` を保持。画像を幅に合わせ拡縮して描画し、縦ラインを重ねる。マウスのクリック/ドラッグで x→時間に変換しシーク。`minimumSize` を小さく申告して壁を作らない |
| `AudioPlayerPanel.kt` | 変更 | `imageLabel` を `TimelineImagePanel` に置換。タイマーで位置更新・シーク連動。開いた時に波形を自動生成。←/→/Home キー追加。設定の読込/保存。ffmpeg 未検出通知。ビジュアライザ表示トグルと下半分の動的な追加/削除。スプリット子要素の最小サイズ緩和 |
| `AudioPlayerSettings.kt` | 変更 | `lastVolume: Int = 80`、`lastLooping: Boolean = false`、`defaultView: String = "waveform"`（waveform/spectrum）、`showVisualizer: Boolean = true` を追加 |
| `AudioAnalyzer.kt` | 変更 | `BufferedImage` を返す生成メソッドを追加（線を重ねるため）。既存の `ImageIcon` 版は `BufferedImage` 版を包む形にして重複を避ける |
| `plugin.xml` | 変更 | 通知グループ（`notificationGroup`）の登録 |

### TimelineImagePanel の責務（インターフェース）

- 入力: `image: BufferedImage?`、`durationMicros: Long`、`positionMicros: Long`、`onSeek: (Long) -> Unit`
- 描画: 画像を幅方向はパネル幅いっぱいに引き伸ばし、高さはパネルの高さに合わせる（アスペクト比は維持しない）。これにより x が常に時間に線形対応する。`durationMicros > 0` のとき `x = positionMicros / durationMicros * width` に縦ラインを描く
- 入力処理: クリック/ドラッグで `time = x / width * durationMicros` を計算し `onSeek(time)` を呼ぶ。範囲外はクランプ
- x↔時間の変換は純粋関数として切り出し、UI 非依存にテスト可能とする
- `getMinimumSize()` は小さい値（例: 幅 50px 程度）を返す

## データフロー・同期

- 再生中: 100ms タイマー → `currentMicroseconds` を `TimelineImagePanel.positionMicros` とシークスライダーの両方へ反映 → ラインとスライダーが同期
- クリック: x → 時間 → `playerService.seek()` → スライダー・ライン更新
- 矢印/Home キー: 現在位置 ±5 秒 / 0 を計算 → `playerService.seek()` → 反映
- 音量/ループ変更時: 即 `AudioPlayerSettings` に保存。次回 `init` で復元（`load` 後に音量を適用）
- ビジュアライザ表示トグル: ON/OFF を `showVisualizer` に保存。OFF で下半分を取り除き波形生成しない。ON で生成

## レイアウトとビジュアライザ表示/非表示（E）

- コントロール部に常時見える「ビジュアライザ表示」トグルを置く
- ON: 既存どおり上下分割で下半分に可視化パネル。可視化内で波形/スペクトラムを切替（`defaultView` を記憶）
- OFF: 下半分の可視化パネルを丸ごと非表示にし、情報＋コントロールだけのコンパクト表示。縦方向も小さくできる
- 表示状態(`showVisualizer`)と表示中ビュー(`defaultView`)を設定に記憶し、次回起動時に復元

## エラー処理・エッジケース

- 総時間が 0（読込失敗）／画像未生成: ラインを描かず、クリックは無視
- ffmpeg 未検出: 画像なし＋通知。各操作は安全に no-op。設定リンクは併存
- リサイズ: 画像はパネル幅に合わせて拡縮、ライン位置は比率で再計算
- 画像より広い余白へのクリック: 範囲内にクランプ
- ビジュアライザ OFF: 波形を生成しない（無駄な ffmpeg 実行を避ける）。再 ON で生成
- 幅が画像より狭い: 画像を縮小表示。最小幅でもライン・クリックは比率で正しく機能

## テスト方針

- `TimelineImagePanel` の x↔時間変換を純粋関数として切り出しユニットテスト（UI 非依存）
- 設定の既定値・永続化（`lastVolume`/`lastLooping`/`defaultView`/`showVisualizer`）のテスト
- 既存テストはそのまま緑を維持

## 実装の順序

1. 設定の永続化（音量・ループ・表示状態）＋ ←/→/Home キー（小さく確実）
2. `TimelineImagePanel`（拡縮＋ライン＋クリックシーク）＋ 波形自動生成 ＋ リサイズの壁を解消（要件 D）
3. ビジュアライザ表示/非表示トグル（要件 E）
4. ffmpeg 未検出通知（要件 C）
